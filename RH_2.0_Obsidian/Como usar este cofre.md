# Como usar este cofre

Este cofre existe pra você **nunca perder o rastro** de uma decisão e conseguir **estudar depois de onde parou**. Ele responde três perguntas:

| Quando você pensar... | Vá para... |
|---|---|
| "Por que decidimos X?" | [[Registro de Decisões]] |
| "Como a parte Y funciona?" | pasta **Arquitetura** (ex.: [[Folha de Pagamento]]) |
| "O que eu preciso estudar pra entender isso?" | [[Cronograma de Estudos]] |
| "O que ainda falta decidir?" | [[Status das Perguntas]] |

## As três camadas de registro

1. **Decisões** — o *porquê*. Cada decisão importante vira uma entrada no [[Registro de Decisões]] no formato **Contexto → Decisão → Consequência → Fonte**. É o que você lê pra defender uma escolha numa entrevista.
2. **Arquitetura** — o *como*. Explica o desenho técnico de cada parte, com o nível de detalhe necessário pra você conseguir corrigir um bug ali.
3. **Estudos** — o *como aprender*. Pra cada tecnologia, o que ela é, por que está no projeto, os fundamentos a dominar e onde estudar.

## Como isso conecta com o código (quando começar)

- Todo **commit** aponta pra nota que o justifica — ver [[Commits]]. Assim, `git log` vira um índice navegável do cofre.
- Ao tomar uma decisão nova durante a implementação, **registre primeiro aqui**, depois code. O cofre nunca fica atrás do código.
- Ao aprender algo que te destravou, adicione uma linha na nota de Estudos correspondente. O cofre cresce com você.

## Links `[[assim]]`

No Obsidian, `[[Nome da Nota]]` cria um link navegável. Abra o **grafo** (ícone de estrela/rede na lateral) pra ver o mapa inteiro de como decisões, arquitetura e estudos se conectam. Um link pra uma nota que ainda não existe aparece "apagado" — é um lembrete do que ainda falta escrever.
