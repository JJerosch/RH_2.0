Bloqueante para começar (Fase 0-1)

  1. Perfis de acesso
  - Qual é a lista fechada de perfis do sistema? (ADMIN, DP_CHEFE, "RH/DP" — são o mesmo perfil ou perfis distintos?)
  - Existe um perfil "Gestor"? Se sim, o que ele pode fazer que RH/DP não pode?
  - Candidato tem conta/login, ou o fluxo é público até a confirmação do RH?

  2. 2FA — parâmetros concretos
  - Quantas tentativas erradas (N) até bloquear?
  - Qual o tempo de expiração do código (o plano cita "ex.: 5 min" só como exemplo)?
  - Existe ação de reenviar código, ou só gerar um novo (invalidando o anterior)?
  - O que acontece depois do bloqueio — espera automática ou desbloqueio manual por admin?

  3. Funcionário — cadastro manual
  - Quais campos são obrigatórios num cadastro manual de funcionário via Filament (fora do fluxo de contratação via vaga)?
  - Fluxo de promoção: quem aprova, o que muda exatamente (cargo, salário, departamento)?

  4. Benefícios
  - TipoBeneficio tem que campos (valor, recorrência, desconta em folha automaticamente)?
  - Como FuncionarioBeneficio entra no GeradorLancamentos da folha — automático todo mês ou ação manual?

  Importante antes da Fase 1-2 (ponto/férias, que alimentam a folha)

  5. Ponto
  - Como uma marcação (PontoMarcacao) se transforma em apuração diária (PontoApuracaoDiaria)? Existe regra de tolerância/jornada?
  - PontoOcorrencia e PontoCalendario servem para quê na prática?

  6. Férias
  - Fluxo de solicitação (FeriasSolicitacao): quem aprova, quantas etapas? Isso passa por Filament (ação administrativa) ou tela própria do funcionário?
  - Como o período aquisitivo (FeriasPeriodoAquisitivo) é atualizado com o tempo (vence, renova)?

  Importante antes da Fase 3 (recrutamento)

  7. Recrutamento — fluxo ponta a ponta
  - Quais etapas exatas uma candidatura percorre (Kanban — mesmas etapas do projeto Spring original)?
  - O que dispara a criação do Funcionario (mudança pra etapa "Contratado", ou outra ação)?
  - O que acontece se o CPF do candidato já existir como funcionário?
  - Depois que o RH confirma os dados extraídos pela IA, a candidatura é criada automaticamente ou precisa de uma ação extra?

  Pode esperar, mas decide o tamanho do trabalho

  8. Folha
  - Qual o nome do tipo "mensal" padrão de FolhaPagamento (só RESCISAO está nomeado)?
  - Reabrir folha sem ser ADMIN/DP_CHEFE: 403 simples, ou mensagem específica?
  - A explicação por IA só pode ser pedida em folha fechada, ou também numa aberta?

  9. Contratos de API
  - Payload/resposta de cada um dos 6 endpoints de folha precisam ser definidos — decide endpoint por endpoint conforme implementa, ou define tudo antes?
  - Os módulos administrativos (funcionarios, organizacao, ponto, ferias, beneficios, solicitacoes) ficam só em Filament (sem API REST própria), ou também precisam de endpoint JSON pra algum consumo
  externo/mobile futuro?

  10. Frontend
  - Lista completa de telas Blade fora de Filament — vai reaproveitar o layout visual do projeto Spring original como referência, ou desenhar do zero?
  - Confirma que Filament cobre todo o CRUD administrativo listado no plano (organizacao, funcionarios, ponto, ferias, beneficios, solicitacoes, relatorios), e que só meus-holerites e o Kanban de recrutamento
  ficam em Blade customizado?

  11. Relatórios
  - Formato de saída — tela Filament (widget/Resource), PDF, CSV?
  - Quais filtros (período, departamento)?
  - Lista fechada definitiva: são exatamente esses 3 (headcount, custo de folha, férias vencendo) ou mais?

  12. App solicitacoes (genérico)
  - O que é esse módulo exatamente? Substitui os fluxos específicos de férias/promoção/desligamento, ou coexiste com eles?

  Decisão de escopo, não de detalhe

  13. Auditoria/LGPD
  - Vale a pena ter log de auditoria de alterações nesse estágio de portfólio (o Spring original tinha um trigger pra isso), ou fica documentado como "fora de escopo por agora"? Em Laravel isso seria via
  pacote (spatie/laravel-activitylog) ou Observer próprio — decisão fica mais barata de implementar que no plano Django, mas ainda não está definida.

  14. Performance/Disponibilidade/Escalabilidade
  - Esse sistema é só demo local, ou tem expectativa real de uso (mesmo pequena)? Decide se vale meta de RNF aqui ou se fica formalmente fora de escopo.

  15. Testes E2E
  - Vale investir em E2E (Playwright, ou o Dusk nativo do Laravel) dado o orçamento de horas já maior que o plano Django (~280-300h), ou aceitar como decisão consciente que fica só unit + feature test por
  agora?

  Pontos específicos da troca de stack (novos, não existiam na lista Django)

  16. Ambiente de PDF
  - spatie/laravel-pdf depende de Node + Chromium instalado no ambiente (via Browsershot). Isso é viável no seu setup local/de deploy, ou prefere cair pra dompdf (mais leve, CSS limitado) como alternativa mais
  simples?

  17. Confiabilidade do cliente de IA sem SDK oficial
  - Como o wrapper Guzzle contra a API da Claude vai tratar retry/timeout/rate limit (o SDK Python fazia isso de fábrica)? Vale escrever essa camada já na Fase 5, ou aceitar uma versão mínima sem retry no MVP?
